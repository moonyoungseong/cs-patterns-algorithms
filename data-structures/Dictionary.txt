Dictionary 핵심 정리정의: 키(key)와 값(value)을 쌍으로 저장하는 해시 기반 맵.Dictionary<TKey, TValue> 형태로 사용.주요특징키로 값을 즉시 조회: 평균 접근 시간 O(1) (상수 시간)키는 유일(unique) 해야 함(중복 불가)순서가 보장되지 않음 (인덱스 기반 접근 아님)주요 메서드Add(key, value) — 추가 (같은 키가 있으면 예외)TryGetValue(key, out value) — 안전 조회 (존재하면 true)ContainsKey(key) — 키 존재 확인Remove(key) — 삭제Clear() — 전체 삭제복잡도조회/추가/삭제: 평균 O(1), 최악 O(n) (해시 충돌 등)언제 쓰나?아이템 ID → 아이템 데이터 (빠른 조회 필요)키(유저ID, 오브젝트ID)로 컴포넌트/프리팹 매핑빈도 센팅, 캐시(이미 로드한 리소스 저장)설정값 맵(설정키→값) 등유니티에서의 주의점Unity의 Inspector는 Dictionary를 기본 직렬화하지 않음.그래서 에디터에서 편하게 편집하려면:SerializableKeyValuePair 리스트로 만들고 에디터 스크립트로 동기화하거나ScriptableObject(아이템 데이터베이스) 사용, 또는SerializableDictionary 같은 외부 라이브러리 사용런타임에서 Dictionary는 매우 유용하지만, 직렬화/인스펙터 표시는 별도 처리 필요.
