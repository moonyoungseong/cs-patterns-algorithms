HashSet<T> 핵심 정리정의HashSet<T>는 중복을 허용하지 않는 집합(Set) 컬렉션이다.내부적으로 해시(Hash) 기반으로 동작하여항목의 존재 여부를 매우 빠르게 확인할 수 있다.같은 값은 한 번만 저장순서 개념 없음주요 연산Add(item) : 항목 추가 (이미 있으면 추가 안 됨)Remove(item) : 항목 제거Contains(item) : 항목 존재 여부 확인Count : 현재 요소 개수Clear() : 전체 비우기복잡도Add / Remove / Contains👉 평균 O(1) (상수 시간)※ 해시 충돌이 심하면 최악의 경우 O(n)이 될 수 있으나실무/일반적인 상황에서는 거의 O(1)로 취급언제 쓰나?중복을 절대 허용하면 안 될 때“이게 이미 있나?”를 자주 검사해야 할 때순서가 중요하지 않은 데이터 집합예:방문 여부 체크이미 처리한 ID 관리이벤트 중복 실행 방지2️⃣ HashSet 장단점 & 사용처장점중복 자동 제거Contains 성능이 매우 빠름 (O(1))방문 체크, 상태 관리에 최적코드 의도가 명확함 (“집합”)단점순서 보장 안 됨인덱스로 접근 불가정렬된 데이터가 필요하면 부적합 (SortedSet, List 사용)커스텀 클래스 사용 시→ Equals, GetHashCode 구현 필요유니티에서 자주 쓰이는 곳 (대표)방문 체크이미 탐색한 노드이미 상호작용한 오브젝트중복 방지이미 수확한 작물 ID한 번만 지급되는 보상트리거 / 충돌 중복 방지같은 오브젝트가 여러 번 Enter 되는 것 방지퀘스트 / 이벤트 상태 관리완료한 퀘스트 ID 집합활성화된 상태/버프 목록HashSet<T> → “중복 없이 존재 여부가 중요”한 줄 요약HashSet<T>는 순서보다 ‘중복 제거 + 빠른 검색’이 중요한 상황에서 쓰는 자료구조
